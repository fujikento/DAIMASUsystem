# 07 運営管理設計書

## 概要

イマーシブダイニングの店舗運営を支援するための予約管理・スタッフ管理・運営ダッシュボードシステムの設計書。

---

## A. 予約管理システム

### 予約フロー

```
Web/電話予約
    ↓
予約作成 (status: confirmed)
    ↓
当日チェックイン (status: checked_in)
    ↓
テーブルアサイン (table_number 設定, status: seated)
    ↓
コース進行 (投影システムと連動)
    ↓
コース完了 (status: completed)
```

### テーブル回転ポリシー

| 回転 | 入店時刻 | 退店目安 | 備考 |
|------|---------|---------|------|
| 1回転目 | 18:00 | 20:00 | コース約2時間 |
| 2回転目 | 19:30 | 21:30 | 重複あり（大テーブル分割時） |
| 3回転目 | 21:00 | 23:00 | 最終回転 |

- タイムスロット: `"18:00"`, `"19:30"`, `"21:00"` の3種類
- 1スロット当たりの最大収容: テーブル数 × テーブル最大人数
- 回転間クリーニング時間: 15〜20分（テーブル投影キャリブレーション含む）

### バースデー / 特別イベント

- 予約時に `special_occasion` フィールドに種別を記録: `birthday`, `anniversary`, `proposal`, `business`, `other`
- `special_occasion = "birthday"` の場合、`birthday_reservations` テーブルとの紐付けを推奨
- `theme_override` で特定日の投影テーマを上書き可能（例: 誕生日ゲスト希望テーマ）
- 特別リクエストは `special_requests` に自由記述

### キャンセル / 変更ポリシー

| 変更タイミング | 対応 |
|--------------|------|
| 3日前以降 | ステータスを `cancelled` に更新（ペナルティなし） |
| 前日 | 50%キャンセル料（運用上の記録のみ。決済は外部システム） |
| 当日 | 100%キャンセル料（運用上の記録のみ） |
| 変更（日時） | `PUT /api/reservations/{id}` でフィールド更新 |

### 予約ステータス遷移

```
confirmed → checked_in → seated → completed
     ↓
  cancelled（どのステータスからも遷移可能）
```

## B. スタッフ管理

### ロール定義

| ロール | 説明 | 主な担当業務 |
|--------|------|------------|
| `manager` | マネージャー | 全権限。予約管理、スタッフ割当、売上確認 |
| `chef` | シェフ | コースメニュー確認、料理タイミング確認 |
| `hall` | ホールスタッフ | 予約チェックイン、テーブルアサイン、ゲスト対応 |
| `projection_operator` | プロジェクションオペレーター | 投影システム操作、コンテンツ切替、AI生成管理 |

### シフト管理

- シフト区分: `opening`（開店準備 17:00〜）、`dinner`（営業 18:00〜23:00）、`closing`（閉店 23:00〜）
- 最小シフト構成: ホール×1、プロジェクションオペレーター×1、マネージャー×1（兼任可）
- シェフはキッチンシステムと別管理（本システムは投影・予約側のみ）

### 権限マトリクス

| 画面 / 操作 | manager | chef | hall | projection_operator |
|------------|---------|------|------|---------------------|
| 運営ダッシュボード | ○ | 閲覧のみ | 閲覧のみ | 閲覧のみ |
| 予約一覧・詳細 | ○ | ○（料理関連のみ） | ○ | 閲覧のみ |
| 予約作成・変更 | ○ | × | ○ | × |
| チェックイン操作 | ○ | × | ○ | × |
| テーブルアサイン | ○ | × | ○ | × |
| 投影システム操作 | ○ | × | × | ○ |
| AI生成ジョブ管理 | ○ | × | × | ○ |
| テーマ・コース設定 | ○ | ○ | × | ○ |
| スタッフ管理 | ○ | × | × | × |
| システム設定 | ○ | × | × | × |

> 認証・認可の実装は `/api/auth` ルーターおよびJWT/セッション管理で別途実装予定。

## C. 運営ダッシュボード

### ダッシュボード構成

```
┌─────────────────────────────────────────────┐
│  本日のサマリーカード（横並び3枚）              │
│  [本日の予約数] [稼働テーブル数] [次の予約まで] │
├────────────────────┬────────────────────────┤
│  テーブルステータス  │  AI生成ジョブステータス  │
│  (テーブルグリッド)  │  (ジョブリスト)         │
├────────────────────┴────────────────────────┤
│  本日の予約タイムライン（横軸=時間、縦軸=テーブル）│
├─────────────────────────────────────────────┤
│  本日の予約一覧（テーブル形式）                │
├─────────────────────────────────────────────┤
│  クイックアクション                           │
│  [新規予約] [ショー開始] [テーマ変更]          │
└─────────────────────────────────────────────┘
```

### テーブルステータス表示

各テーブルを以下の状態で色分け表示:

| ステータス | 色 | 説明 |
|-----------|-----|------|
| 空席 | グレー | 予約なし / 準備完了 |
| チェックイン済 | 黄色 | ゲスト到着、テーブル案内待ち |
| 着席中 | 緑 | コース進行中 |
| 片付け中 | オレンジ | コース完了、次の回転準備中 |
| 予約あり | ブルー | 当日予約は入っているが未着 |

### AI生成ジョブステータス

- `Storyboard` の各シーンの `image_status` / `video_status` を集計して表示
- ステータス: `pending`（待機）, `processing`（生成中）, `complete`（完了）, `failed`（失敗）
- 失敗ジョブは再実行ボタンを表示

### システムヘルス

| コンポーネント | 確認方法 |
|--------------|---------|
| FastAPI | `GET /` のレスポンス確認 |
| データベース | DB接続確認 |
| 投影システム | WebSocket接続状態 (`ws://host/api/projection/ws`) |
| プロジェクター | `GET /api/projection/status` の `state` フィールド |

### デザイン仕様

- カラースキーム: ダークテーマ
  - 背景: `bg-[#080f1a]`（最外）、`bg-[#0e1d32]`（カード背景）
  - ボーダー: `border-blue-400/[0.10]`
  - テキスト: ホワイト系 (`text-white`, `text-neutral-300`, `text-neutral-500`)
  - アクセント: ブルー系 (`text-blue-400`, `bg-blue-600`)
- レスポンシブ: デスクトップ（1280px〜）を主とし、タブレット（768px〜）でも使用可能

## D. APIエンドポイント設計

### 予約管理 `/api/reservations`

| メソッド | パス | 説明 |
|---------|------|------|
| POST | `/api/reservations` | 予約作成 |
| GET | `/api/reservations` | 予約一覧（クエリ: `date`, `status`） |
| GET | `/api/reservations/today` | 本日の予約一覧 |
| GET | `/api/reservations/calendar` | 月間カレンダー用（クエリ: `year`, `month`） |
| GET | `/api/reservations/{id}` | 予約詳細 |
| PUT | `/api/reservations/{id}` | 予約変更 |
| DELETE | `/api/reservations/{id}` | キャンセル（ソフトデリート: status→cancelled） |
| POST | `/api/reservations/{id}/check-in` | チェックイン処理 |

### データモデル

```python
class Reservation(Base):
    __tablename__ = "reservations"
    id: int (PK)
    guest_name: str
    guest_email: str | None
    guest_phone: str | None
    party_size: int
    reservation_date: date
    time_slot: str          # "18:00" | "19:30" | "21:00"
    table_number: int | None
    status: str             # confirmed | checked_in | seated | completed | cancelled
    special_occasion: str | None  # birthday | anniversary | proposal | business | other
    special_requests: text | None
    theme_override: str | None
    created_at: datetime
    updated_at: datetime
```
