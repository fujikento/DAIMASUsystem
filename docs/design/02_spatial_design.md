### 1-3. エッジブレンド 120px 詳細仕様

```
  PJ1 出力 (1920px)      PJ2 出力 (1920px)      PJ3 出力 (1920px)
  ┌───────────────────┐   ┌───────────────────┐   ┌───────────────────┐
  │                 ▓▓│   │▓▓                ▓▓│   │▓▓                 │
  │   有効領域      ▓▓│   │▓▓  有効領域      ▓▓│   │▓▓   有効領域      │
  │   (1800px)      ▓▓│   │▓▓  (1680px)      ▓▓│   │▓▓   (1800px)      │
  │                 ▓▓│   │▓▓                ▓▓│   │▓▓                 │
  └───────────────────┘   └───────────────────┘   └───────────────────┘
                  ↑ 120px ↑           ↑ 120px ↑
                  ブレンド領域         ブレンド領域

  ブレンド計算:
  - PJ1: x=0     → x=1920  (右端 120px がブレンド)
  - PJ2: x=1800  → x=3720  (左端 120px + 右端 120px がブレンド)
  - PJ3: x=3600  → x=5520  (左端 120px がブレンド)

  ブレンド式（輝度の線形補間）:
  overlap_region において PJ_n の輝度 = 1.0 - (x / 120)
  overlap_region において PJ_{n+1} の輝度 = x / 120
  （x: ブレンド領域内の水平位置 0〜120）
```

## 2. 投影モード仕様

### 2-1. unified（統一モード）— 5520×1200 パノラマ全体投影

```
  用途: テーブル全体を1つのシームレスな映像で覆う
  解像度: 5,520 × 1,200 px
  アスペクト比: 4.6:1 (超ワイド)

  生成フロー:
    Runway 21:9 (3840×1080) × 2セグメント（Left/Right）
    → 各セグメントを縦方向スケール: 1080→1200px (+11%)
    → 横方向スケール後: 4,267 × 1,200 px
    → 左右20%オーバーラップでクロスフェードブレンド合成
    → 最終出力: 5,520 × 1,200 px

  OSC制御:
    /unified/play    [file_path: str]  — 全体映像の再生
    /unified/brightness [value: float]  — 全体輝度 0.0〜1.0

  適用シーン:
    - 海や森などの広大な自然映像
    - コース全体を包む統一的な演出
    - テーブル全体を流れる液体・光の表現
```

### 2-2. zone（ゾーンモード）— 1380×1200 個別ゾーン投影

```
  用途: 各席エリアごとに異なる映像を独立再生
  解像度: 1,380 × 1,200 px / ゾーン
  アスペクト比: 1.15:1 (ほぼ正方形)

  生成フロー:
    Runway 1:1 (2160×2160) 正方形映像
    → 幅1380pxにスケールダウン: 2160→1380 (高さ: 2160→1380)
    → 中央クロップ: 1380×1200px に切り出し

  OSC制御:
    /zone/{id}/play        [file_path: str]  — ゾーン映像再生
    /zone/{id}/brightness  [value: float]    — 個別輝度制御
    /zone/{id}/stop        []                — ゾーン停止

  適用シーン:
    - 各席の料理に合わせた個別演出
    - 席ごとに異なるアレルギー対応演出
    - 誕生日席のみ特別演出
```

### 2-3. custom（カスタムモード）— 任意ゾーン組み合わせ投影

```
  用途: 複数の特定ゾーンを選択して同一映像を投影
  対象ゾーン: 1〜4の任意の組み合わせ（例: "1,3" や "2,3,4"）

  組み合わせ例:
    "1,2"   → Zone1+Zone2 で2760×1200px の半分パノラマ
    "2,3"   → 中央2ゾーンの集中演出
    "1,2,3,4" → unified相当（全ゾーン）

  座標変換:
    選択ゾーンが連続する場合: 結合して1つのキャンバスに投影
    選択ゾーンが非連続の場合: 各ゾーンに同一映像を独立投影

  OSC制御:
    /custom/set_zones  [zone_ids: str]  — 対象ゾーンをカンマ区切りで指定
    /custom/play       [file_path: str] — カスタム投影開始

  Pydantic スキーマ例:
    target_zones: "1,2,3,4"  (StoryboardScene.target_zones と同形式)
```

### 2-4. transition（トランジションエフェクト）

| トランジション名 | 説明 | 推奨duration | OSCアドレス |
|----------------|------|-------------|-------------|
| crossfade | 透明度の線形クロスフェード | 1.5〜3.0秒 | /transition crossfade {duration} |
| cut | 即時切替（フレーム単位） | 0.0秒 | /transition cut 0 |
| fade_black | 黒にフェードアウト→次映像イン | 2.0〜4.0秒 | /transition fade_black {duration} |
| fade_white | 白フラッシュ→次映像イン | 1.0〜2.0秒 | /transition fade_white {duration} |
| wipe_left | 左から右へワイプ | 1.0〜2.0秒 | /transition wipe_left {duration} |
| wipe_right | 右から左へワイプ | 1.0〜2.0秒 | /transition wipe_right {duration} |
| sparkle | キラキラパーティクル演出 | 2.0〜5.0秒 | /transition sparkle {duration} |
| ripple | 波紋エフェクト | 2.0〜3.0秒 | /transition ripple {duration} |
| zoom_in | ズームインしながら切替 | 1.5〜2.5秒 | /transition zoom_in {duration} |

ゾーン間トランジション（zone_from / zone_to 指定）:
```
  /zone_transition {from_zone} {to_zone} {type} {duration}
  例: /zone_transition 2 3 crossfade 2.0
  （Zone2の映像をフェードアウトしながらZone3に展開）
```

### 3-1. 通信設定

| 項目 | 値 |
|---|---|
| プロトコル | OSC over UDP |
| TouchDesigner 受信ホスト | 127.0.0.1 |
| TouchDesigner 受信ポート | 7000 |
| API→TD 方向 | Python → TouchDesigner |
| TD→API 方向 | TouchDesigner → Python (ポート 7001) ※将来対応 |

### 3-2. コマンド一覧

#### 再生制御

| OSCアドレス | 引数 | 型 | 範囲 | 説明 |
|------------|------|----|------|------|

| `/play` | timeline_id, table_id | int, str | — | テーブルIDを指定して再生 |
| `/pause` | — | — | — | 再生一時停止 |
| `/stop` | — | — | — | 再生停止・待機状態へ |

#### コンテンツ制御

| OSCアドレス | 引数 | 型 | 範囲 | 説明 |

#### ゾーン制御（新規）

| OSCアドレス | 引数 | 型 | 範囲 | 説明 |

| `/zone/stop` | zone_id | int | 1〜4 | ゾーン停止 |

#### バースデー演出

| OSCアドレス | 引数 | 型 | 範囲 | 説明 |

#### プリセット（新規）

| OSCアドレス | 引数 | 型 | 範囲 | 説明 |

### 3-3. パラメータ型・範囲一覧

| パラメータ名 | 型 | 有効範囲 | デフォルト | 説明 |
|------------|-----|----------|----------|------|

### 3-4. レスポンスメッセージ（TD→API）

TouchDesignerからのレスポンスは将来的にOSCで受信（ポート7001）:

| OSCアドレス | 引数 | 説明 |
|------------|------|------|
| `/status` | state | str: "idle"\|"playing"\|"paused" |
| `/status/content` | file_path | 現在再生中コンテンツパス |
| `/ack` | command, result | コマンド確認応答 (result: 0=OK, 1=Error) |
| `/error` | code, message | エラー通知 |

### 3-5. エラーハンドリング

```
  エラーコード一覧:
  - 1001: ファイルが見つからない (file_not_found)
  - 1002: 無効なゾーンID (invalid_zone)
  - 1003: 無効なトランジションタイプ (invalid_transition)
  - 1004: TouchDesigner 接続失敗 (connection_failed)
  - 1005: プリセットが見つからない (preset_not_found)

  OSCControllerのエラー処理:
  1. _send() が False を返した場合 → logger.error でログ記録
  2. APIレスポンスは 200 を返しつつ OSC失敗をレスポンスボディに含める
  3. 接続タイムアウト: UDP のため即時返却 (エラーは非同期)
  4. リトライ: 重要なコマンド（play/birthday）は最大3回リトライ（将来対応）
```
